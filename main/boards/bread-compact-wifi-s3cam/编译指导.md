# bread-compact-wifi-s3cam 编译指导

## 一、编译系统如何选择板子配置

### 1. 配置流程（从配置到编译）

```
menuconfig选择 → Kconfig定义 → CMakeLists.txt映射 → 加载板子源文件
```

### 2. 详细步骤解析

#### 步骤1：Kconfig 定义板子选项
**文件：** `main/Kconfig.projbuild` (第73-75行)

```c
config BOARD_TYPE_BREAD_COMPACT_WIFI_CAM
    bool "面包板新版接线（WiFi）+ LCD + Camera"
    depends on IDF_TARGET_ESP32S3
```

这定义了在 `menuconfig` 中显示的选项名称。

#### 步骤2：CMakeLists.txt 映射到目录名
**文件：** `main/CMakeLists.txt` (第209-210行)

```cmake
elseif(CONFIG_BOARD_TYPE_BREAD_COMPACT_WIFI_CAM)
    set(BOARD_TYPE "bread-compact-wifi-s3cam")
```

当选择了 `BOARD_TYPE_BREAD_COMPACT_WIFI_CAM` 配置时，CMake 会将 `BOARD_TYPE` 变量设置为 `"bread-compact-wifi-s3cam"`。

#### 步骤3：加载板子源文件
**文件：** `main/CMakeLists.txt` (第222-226行)

```cmake
file(GLOB BOARD_SOURCES
    ${CMAKE_CURRENT_SOURCE_DIR}/boards/${BOARD_TYPE}/*.cc
    ${CMAKE_CURRENT_SOURCE_DIR}/boards/${BOARD_TYPE}/*.c
)
list(APPEND SOURCES ${BOARD_SOURCES})
```

这会从 `main/boards/bread-compact-wifi-s3cam/` 目录加载所有 `.cc` 和 `.c` 文件。

**实际路径：**
```
${CMAKE_CURRENT_SOURCE_DIR}/boards/bread-compact-wifi-s3cam/*.cc
= main/boards/bread-compact-wifi-s3cam/*.cc
```

#### 步骤4：包含板子头文件
板子的 `config.h` 文件会被包含在源文件中，定义引脚配置。

## 二、编译步骤

### 方法1：使用 menuconfig（推荐）

#### 1. 设置编译目标
```bash
cd /home/zexuan/X/lab/xiaozhi-esp32
idf.py set-target esp32s3
```

#### 2. 打开配置菜单
```bash
idf.py menuconfig
```

#### 3. 选择板子类型
在 menuconfig 中导航：
```
Xiaozhi Assistant 
  └─> Board Type 
       └─> [*] 面包板新版接线（WiFi）+ LCD + Camera
```

**操作：**
- 使用方向键导航
- 按空格键选择（`[*]` 表示已选中）
- 按 `S` 保存配置
- 按 `Q` 退出

#### 4. 编译
```bash
idf.py build
```

#### 5. 烧录
```bash
idf.py -p /dev/ttyUSB0 flash
```

---

### 方法2：直接修改 sdkconfig（快速方法）

#### 1. 编辑 sdkconfig 文件
```bash
cd /home/zexuan/X/lab/xiaozhi-esp32
```

找到以下行并修改：
```bash
# 将这行：
CONFIG_BOARD_TYPE_BREAD_COMPACT_WIFI=y

# 改为：
# CONFIG_BOARD_TYPE_BREAD_COMPACT_WIFI is not set
CONFIG_BOARD_TYPE_BREAD_COMPACT_WIFI_CAM=y
```

#### 2. 设置编译目标（如果还没设置）
```bash
idf.py set-target esp32s3
```

#### 3. 编译
```bash
idf.py build
```

#### 4. 烧录
```bash
idf.py -p /dev/ttyUSB0 flash
```

---

### 方法3：使用 sed 命令快速修改（自动化）

```bash
cd /home/zexuan/X/lab/xiaozhi-esp32

# 备份当前配置
cp sdkconfig sdkconfig.backup

# 修改配置
sed -i 's/^CONFIG_BOARD_TYPE_BREAD_COMPACT_WIFI=y$/# CONFIG_BOARD_TYPE_BREAD_COMPACT_WIFI is not set\nCONFIG_BOARD_TYPE_BREAD_COMPACT_WIFI_CAM=y/' sdkconfig

# 设置目标
idf.py set-target esp32s3

# 编译
idf.py build

# 烧录
idf.py -p /dev/ttyUSB0 flash
```

## 三、验证配置是否正确

### 1. 检查 sdkconfig 文件
```bash
grep "BOARD_TYPE_BREAD_COMPACT_WIFI_CAM" sdkconfig
```

应该看到：
```
CONFIG_BOARD_TYPE_BREAD_COMPACT_WIFI_CAM=y
```

### 2. 检查编译输出
编译时应该看到加载了板子文件：
```
[1/xxx] Building CXX object main/CMakeFiles/__idf_main.dir/boards/bread-compact-wifi-s3cam/compact_wifi_board_s3cam.cc.obj
```

### 3. 检查生成的固件
```bash
ls -lh build/xiaozhi.bin
```

## 六、常见问题

### Q1: 如何确认当前使用的是哪个板子配置？
```bash
grep "CONFIG_BOARD_TYPE.*=y" sdkconfig
```

### Q2: 编译时找不到板子文件？
检查 `sdkconfig` 中是否正确设置了 `CONFIG_BOARD_TYPE_BREAD_COMPACT_WIFI_CAM=y`

### Q3: 如何切换回其他板子？
在 menuconfig 中选择其他板子，或直接修改 `sdkconfig` 文件。
