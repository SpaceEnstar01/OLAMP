## 一、音频引脚配置（I2S Simplex模式）

### 数字麦克风（INMP441 或 ICS43434）
使用独立的I2S接口：

```c
#define AUDIO_I2S_MIC_GPIO_WS   GPIO_NUM_1    // Word Select (LRCLK) - 引脚 1
#define AUDIO_I2S_MIC_GPIO_SCK  GPIO_NUM_2    // Serial Clock (BCLK) - 引脚 2
#define AUDIO_I2S_MIC_GPIO_DIN  GPIO_NUM_42   // Data Input (SD) - 引脚 42
```

**麦克风连接：**
- INMP441/ICS43434 的 WS/LR  → ESP32-S3 GPIO 1
- INMP441/ICS43434 的 SCK/BCLK → ESP32-S3 GPIO 2
- INMP441/ICS43434 的 SD/DIN → ESP32-S3 GPIO 42
- INMP441/ICS43434 的 VDD → 3.3V
- INMP441/ICS43434 的 GND → GND

### 功放（MAX98357A）
使用独立的I2S接口：

```c
#define AUDIO_I2S_SPK_GPIO_DOUT GPIO_NUM_39   // Data Output - 引脚 39
#define AUDIO_I2S_SPK_GPIO_BCLK GPIO_NUM_40   // Bit Clock - 引脚 40
#define AUDIO_I2S_SPK_GPIO_LRCK GPIO_NUM_41  // Left/Right Clock - 引脚 41
```

**MAX98357A 连接：**
- MAX98357A 的 DIN → ESP32-S3 GPIO 39
- MAX98357A 的 BCLK → ESP32-S3 GPIO 40
- MAX98357A 的 LRC → ESP32-S3 GPIO 41
- MAX98357A 的 VIN → 5V（或3.3V，根据模块）
- MAX98357A 的 GND → GND
- MAX98357A 的 OUT+ / OUT- → 喇叭（8Ω 2~3W 或 4Ω 2~3W）

## 二、摄像头引脚配置（OV3660/OV2640）

```c
//Camera Config
#define CAMERA_PIN_D0 GPIO_NUM_11    // 数据位 0 - 引脚 11
#define CAMERA_PIN_D1 GPIO_NUM_9     // 数据位 1 - 引脚 9
#define CAMERA_PIN_D2 GPIO_NUM_8     // 数据位 2 - 引脚 8
#define CAMERA_PIN_D3 GPIO_NUM_10    // 数据位 3 - 引脚 10
#define CAMERA_PIN_D4 GPIO_NUM_12    // 数据位 4 - 引脚 12
#define CAMERA_PIN_D5 GPIO_NUM_18    // 数据位 5 - 引脚 18
#define CAMERA_PIN_D6 GPIO_NUM_17    // 数据位 6 - 引脚 17
#define CAMERA_PIN_D7 GPIO_NUM_16    // 数据位 7 - 引脚 16
#define CAMERA_PIN_XCLK GPIO_NUM_15  // 外部时钟 - 引脚 15
#define CAMERA_PIN_PCLK GPIO_NUM_13  // 像素时钟 - 引脚 13
#define CAMERA_PIN_VSYNC GPIO_NUM_6  // 垂直同步 - 引脚 6
#define CAMERA_PIN_HREF GPIO_NUM_7   // 水平参考 - 引脚 7
#define CAMERA_PIN_SIOC GPIO_NUM_5   // SCCB 时钟 (I2C SCL) - 引脚 5
#define CAMERA_PIN_SIOD GPIO_NUM_4   // SCCB 数据 (I2C SDA) - 引脚 4
#define CAMERA_PIN_PWDN GPIO_NUM_NC // 电源关闭（未使用）
#define CAMERA_PIN_RESET GPIO_NUM_NC // 复位（未使用）
#define XCLK_FREQ_HZ 20000000        // XCLK 频率 20MHz
```

**摄像头连接（OV3660/OV2640）：**
- D0 → GPIO 11
- D1 → GPIO 9
- D2 → GPIO 8
- D3 → GPIO 10
- D4 → GPIO 12
- D5 → GPIO 18
- D6 → GPIO 17
- D7 → GPIO 16
- XCLK → GPIO 15
- PCLK → GPIO 13
- VSYNC → GPIO 6
- HREF → GPIO 7
- SIOC (SCL) → GPIO 5
- SIOD (SDA) → GPIO 4
- 3.3V → 摄像头 VCC
- GND → 摄像头 GND

**注意：** 根据README，摄像头占用了ESP32-S3的USB引脚（GPIO 19和20），这些引脚被用于LCD显示。

## 三、LCD显示引脚配置

```c
#define DISPLAY_BACKLIGHT_PIN GPIO_NUM_38  // 背光控制 - 引脚 38
#define DISPLAY_MOSI_PIN      GPIO_NUM_20  // SPI MOSI - 引脚 20（占用USB引脚）
#define DISPLAY_CLK_PIN       GPIO_NUM_19  // SPI CLK - 引脚 19（占用USB引脚）
#define DISPLAY_DC_PIN        GPIO_NUM_47  // 数据/命令 - 引脚 47
#define DISPLAY_RST_PIN       GPIO_NUM_21  // 复位 - 引脚 21
#define DISPLAY_CS_PIN        GPIO_NUM_45  // 片选 - 引脚 45
```

**LCD连接（SPI接口）：**
- MOSI → GPIO 20
- CLK/SCK → GPIO 19
- DC → GPIO 47
- RST → GPIO 21
- CS → GPIO 45
- BL (背光) → GPIO 38
- VCC → 3.3V 或 5V（根据LCD）
- GND → GND

## 六、引脚冲突说明

⚠️ **重要：** 根据README说明，因为摄像头占用IO较多，占用了ESP32-S3的USB引脚（GPIO 19和20），这两个引脚被用于LCD的SPI通信。

这意味着：
- **GPIO 19** → LCD CLK（SPI时钟）
- **GPIO 20** → LCD MOSI（SPI数据）
- USB功能可能不可用或受限
